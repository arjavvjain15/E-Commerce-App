# AWS S3 Integration & Code Changes Guide

This document outlines the codebase changes made to both the frontend (UI) and backend to support uploading actual images instead of pasting manual URLs, along with a detailed setup guide for your AWS S3 bucket.

---

## 1. Backend Changes (Image Upload Engine)

To support actual file uploads rather than simple text URLs, the following additions were made to the backend:

### A. Dependencies Added
We installed two core packages on the backend:
- `multer`: Middleware to parse incoming multipart form data (`multipart/form-data`) and handle file uploads.
- `@aws-sdk/client-s3`: The official AWS SDK (v3) client for interaction with S3 buckets.

### B. Upload Service File Created
- **File**: `backend/src/services/upload.service.js`
- **Implementation**:
  - `uploadToS3(file)`: Configures the `S3Client`, generates a unique name using `uuidv4`, reads the file buffer, and triggers a `PutObjectCommand` to stream it directly to AWS S3. It returns the hosted S3 link.
  - `uploadLocal(file)`: Serves as a fallback. Creates a local folder named `uploads/` in the backend root directory, writes the buffer locally, and returns the address of the local static file.

### C. Controller & Endpoint Logic
- **File**: `backend/src/controllers/upload.controller.js`
- **Implementation**:
  - `uploadImage`: Validates if a file exists, checks whether S3 configuration variables are defined in the environment (`.env`), and executes either `uploadToS3` or `uploadLocal`.
- **File**: `backend/src/routes/upload.routes.js`
- **Implementation**:
  - Exposes the route `POST /api/upload`.
  - Protected by `authMiddleware` and `roleMiddleware(["admin"])` so only admins can execute uploads.
- **File**: `backend/src/routes/index.js`
  - Registered the `/upload` router.

### D. Static Local Fallback
- **File**: `backend/src/app.js`
- **Implementation**:
  - Added `app.use("/uploads", express.static("uploads"))` so that if S3 is not configured, the local server serves stored files at `http://localhost:3000/uploads/<filename>`.

## 2. Frontend Changes (Admin UI)

The Admin panel was restructured to accept raw image files, display instant preview thumbnails, and perform sequential uploading before form submission.

### A. React State Variables
- **File**: `frontend/src/pages/Admin.jsx`
- **Implementation**:
  - `selectedFile`: Stores the raw image file selected by the user.
  - `imagePreview`: Stores the base64 URL of the chosen image to display a live thumbnail instantly.
  - `imageSourceType`: A toggle ("file" or "url") to switch between uploading a local file or entering an image URL manually.

### B. File Picker & Preview UI
- **File**: `frontend/src/pages/Admin.jsx` (Modal body)
- **Implementation**:
  - Implemented a tabbed interface.
  - Added a dashed upload zone:
    ```jsx
    <div style={{ display: "flex", flexDirection: "column", alignItems: "center", justifyContent: "center", border: "2px dashed var(--border)", borderRadius: "8px", padding: "24px", cursor: "pointer", textAlign: "center" }} onClick={() => document.getElementById("product-image-file").click()}>
      <input type="file" id="product-image-file" accept="image/*" style={{ display: "none" }} onChange={handleFileChange} />
      <span>📁 Click to select an image file</span>
    </div>
    ```
  - Added a live image preview box that displays the selected file (via `FileReader` base64 encoder) or the typed URL.

### C. Form Submit Integration
- **File**: `frontend/src/pages/Admin.jsx` (`handleProductSubmit`)
- **Implementation**:
  - Before calling the product creation endpoint, if `imageSourceType` is set to `"file"` and `selectedFile` is present, it constructs a `FormData` object and fires an API request to `POST /api/upload`.
  - Once the backend responds with the hosted URL (either S3 or local fallback), it replaces the image link in the product payload and creates/updates the product.

## 3. Step-by-Step AWS S3 Configuration

### Step 1: Create the AWS S3 Bucket
1. Open the [AWS S3 Console](https://console.aws.amazon.com/s3/).
2. Click **Create bucket**.
3. **Bucket name**: Enter a unique name (e.g., `my-ecommerce-product-images`).
4. **AWS Region**: Select the region closest to your users (e.g., `ap-south-1` for Mumbai). *Make note of this region identifier.*
5. **Object Ownership**: Choose **ACLs disabled**.
6. **Block Public Access settings for this bucket**:
   - **Uncheck** "Block *all* public access".
   - Check the acknowledgement warning box underneath. (This allows product images to be publicly readable by shoppers).
7. Click **Create bucket**.

### Step 2: Configure CORS (Cross-Origin Resource Sharing)
To allow your frontend application to fetch/upload resources to S3 from a web browser:
1. In S3, click your new bucket name.
2. Go to the **Permissions** tab.
3. Scroll to **Cross-origin resource sharing (CORS)** and click **Edit**.
4. Paste the following configuration:
   ```json
   [
       {
           "AllowedHeaders": ["*"],
           "AllowedMethods": ["GET", "HEAD"],
           "AllowedOrigins": ["*"],
           "ExposeHeaders": []
       }
   ]
   ```
5. Click **Save changes**.

### Step 3: Configure Bucket Policy (For Public Read Access)
This ensures everyone can view the product images:
1. In the same **Permissions** tab, find **Bucket policy** and click **Edit**.
2. Paste the following policy, replacing `YOUR_BUCKET_NAME` with your actual bucket name:
   ```json
   {
       "Version": "2012-10-17",
       "Statement": [
           {
               "Sid": "PublicReadGetObject",
               "Effect": "Allow",
               "Principal": "*",
               "Action": "s3:GetObject",
               "Resource": "arn:aws:s3:::YOUR_BUCKET_NAME/*"
           }
       ]
   }
   ```
3. Click **Save changes**.

### Step 4: Create an IAM User (Get Credentials)
1. Open the [IAM Console](https://console.aws.amazon.com/iam/).
2. Click **Users** in the left menu, then click **Create user**.
3. **User name**: e.g., `s3-uploader-user`. Click **Next**.
4. **Permissions options**: Choose **Attach policies directly**.
5. Search for and check **`AmazonS3FullAccess`**.
6. Click **Next**, then **Create user**.
7. Click on your newly created user from the list.
8. Go to the **Security credentials** tab.
9. Scroll down to **Access keys** and click **Create access key**.
10. Select **Local code** (or **Application running outside AWS**), check the confirmation checkbox, and click **Next**.
11. Click **Create access key**.
12. **Copy the Access key ID and Secret access key** (save them securely, as they will only be shown once).

### Step 5: Update Backend Environment
Paste the credentials into your `backend/.env` file:
```env
AWS_ACCESS_KEY_ID=your_access_key_id
AWS_SECRET_ACCESS_KEY=your_secret_access_key
AWS_REGION=your_aws_region_code (e.g. ap-south-1)
AWS_S3_BUCKET=your_bucket_name
```
Restart the backend dev server, and all product uploads will now be stored in your AWS S3 bucket!
